# Одна методика картирования знаний 

- https://youtu.be/7cuj7dp8dhk?si=Bw25hVtBfj3yPNQk
- https://knowledgeconf.ru/2020/abstracts/6442

---

#### **Основная идея:**
Картирование знаний — это процесс и методика, которая помогает выявить и структурировать экспертные знания сотрудников, чтобы эффективно управлять ими в организации.

---

#### **Ключевые тезисы:**

1. **Что такое картирование знаний?**
   - Это процесс выявления областей экспертизы сотрудников и критически важных знаний в организации.
   - Позволяет определить скрытый потенциал команды, распределить экспертные нагрузки и выявить ключевых сотрудников.

2. **Зачем это нужно?**
   - Для определения скрытого потенциала команды.
   - Для разгрузки загруженных экспертов.
   - Для принятия решений в кризисных ситуациях (например, кого увольнять).
   - Для поиска внутренних тренеров и определения направлений развития сотрудников.

3. **Основные понятия:**
   - **Знание организации** — совокупность экспертных знаний сотрудников, зафиксированных или незафиксированных на корпоративных носителях.
   - **Компетенция** — интегральная способность, объединяющая знания, навыки и ценностные установки.

4. **Примеры и кейсы:**
   - Пример с водителями грузовиков: одинаковый опыт, но разные ценностные установки приводят к разным результатам.
   - Кейс подбора команды для проекта по цифровому анализу породы: выявление кросс-функциональных экспертов.

5. **Методы сбора данных:**
   - Опросники для самооценки сотрудников.
   - Анализ взаимодействий между сотрудниками (например, кто к кому обращается за советом).
   - Использование инструментов анализа графов (например, Gephi).

6. **Анализ результатов:**
   - **Метрики анализа графов:** степень узла, диаметр сети, плотность, средняя длина пути.
   - **Центральность:** выявление ключевых экспертов и людей, влияющих на передачу информации.
   - **Пример анализа графа:** выявление кластеров сотрудников и ключевых экспертов.

7. **Практическое применение:**
   - Оптимизация рассадки сотрудников в офисе для улучшения взаимодействия.
   - Выявление рисков утраты знаний (например, из-за пенсионного возраста или опасных хобби).
   - Поддержка и развитие ключевых экспертов.

8. **Сложности и ограничения:**
   - Анализ свободного ввода данных требует значительных ресурсов.
   - Мотивация сотрудников на заполнение опросников.
   - Вопросы безопасности и конфиденциальности данных.

9. **Рекомендации:**
   - Начинать с небольших подразделений, а не с компании в целом.
   - Использовать комбинацию свободного ввода и выбора из вариантов для упрощения анализа.
   - Периодически обновлять данные, чтобы избежать устаревания информации.

---

#### **Вывод:**
Картирование знаний — это мощный инструмент для управления знаниями в организации, который помогает выявить ключевых экспертов, оптимизировать процессы и снизить риски утраты критически важных знаний. Однако его внедрение требует тщательной подготовки, анализа и поддержки со стороны руководства.

### Метрики анализа графа взаимодействий

При анализе графа взаимодействий между сотрудниками используются различные метрики, которые помогают выявить ключевых сотрудников, "узкие места" и структуру сети. Ниже приведено детальное описание основных метрик.

---

#### **1. Степень узла (Degree Centrality)**
- **Определение:** Количество связей (ребер), которые имеет узел (сотрудник).
- **Формула:**  
  \[
  \text{Degree Centrality}(v) = \frac{\text{Количество связей узла } v}{\text{Общее количество узлов} - 1}
  \]
- **Интерпретация:**
  - Высокая степень узла указывает на активного сотрудника, который часто взаимодействует с другими.
  - Низкая степень узла может указывать на изолированного сотрудника.
- **Пример:** Если сотрудник А взаимодействует с 4 из 10 сотрудников, его степень узла равна 0.4.

---

#### **2. Центральность по близости (Closeness Centrality)**
- **Определение:** Мера того, насколько близко узел находится ко всем другим узлам в графе.
- **Формула:**  
  \[
  \text{Closeness Centrality}(v) = \frac{1}{\sum_{u \neq v} d(u, v)}
  \]
  где \(d(u, v)\) — кратчайший путь между узлами \(u\) и \(v\).
- **Интерпретация:**
  - Высокая центральность по близости указывает на сотрудника, который может быстро распространять информацию.
  - Низкая центральность по близости указывает на сотрудника, который находится "на периферии" сети.
- **Пример:** Если сотрудник Б находится в среднем на расстоянии 2 шагов от всех остальных, его центральность по близости равна 0.5.

---

#### **3. Центральность по посредничеству (Betweenness Centrality)**
- **Определение:** Мера того, как часто узел выступает "мостом" на кратчайших путях между другими узлами.
- **Формула:**  
  \[
  \text{Betweenness Centrality}(v) = \sum_{s \neq v \neq t} \frac{\sigma_{st}(v)}{\sigma_{st}}
  \]
  где:
  - \(\sigma_{st}\) — количество кратчайших путей между узлами \(s\) и \(t\).
  - \(\sigma_{st}(v)\) — количество кратчайших путей между \(s\) и \(t\), которые проходят через узел \(v\).
- **Интерпретация:**
  - Высокая центральность по посредничеству указывает на сотрудника, который контролирует поток информации.
  - Низкая центральность по посредничеству указывает на сотрудника, который не играет ключевой роли в передаче информации.
- **Пример:** Если сотрудник В находится на кратчайших путях между многими парами сотрудников, его центральность по посредничеству будет высокой.

---

#### **4. Центральность по собственному вектору (Eigenvector Centrality)**
- **Определение:** Мера влияния узла, учитывающая не только количество связей, но и важность связанных узлов.
- **Формула:**  
  \[
  \text{Eigenvector Centrality}(v) = \frac{1}{\lambda} \sum_{u \in G} A_{vu} \cdot \text{Eigenvector Centrality}(u)
  \]
  где:
  - \(A_{vu}\) — элемент матрицы смежности (1, если есть связь между \(v\) и \(u\), иначе 0).
  - \(\lambda\) — наибольшее собственное значение матрицы смежности.
- **Интерпретация:**
  - Высокая центральность по собственному вектору указывает на сотрудника, связанного с другими влиятельными сотрудниками.
  - Низкая центральность по собственному вектору указывает на сотрудника, связанного с менее влиятельными коллегами.
- **Пример:** Если сотрудник Г связан с ключевыми экспертами, его центральность по собственному вектору будет высокой.

---

#### **5. Плотность графа (Graph Density)**
- **Определение:** Мера того, насколько плотно связаны узлы в графе.
- **Формула:**  
  \[
  \text{Density} = \frac{2 \cdot \text{Количество ребер}}{\text{Количество узлов} \cdot (\text{Количество узлов} - 1)}
  \]
- **Интерпретация:**
  - Высокая плотность указывает на тесное взаимодействие между сотрудниками.
  - Низкая плотность указывает на слабые связи и возможные "разрывы" в сети.
- **Пример:** Если в графе 10 узлов и 20 ребер, плотность графа равна 0.44.

---

#### **6. Диаметр графа (Graph Diameter)**
- **Определение:** Максимальное расстояние между двумя узлами в графе.
- **Формула:**  
  \[
  \text{Diameter} = \max(d(u, v))
  \]
  где \(d(u, v)\) — кратчайший путь между узлами \(u\) и \(v\).
- **Интерпретация:**
  - Большой диаметр указывает на слабую связность сети.
  - Малый диаметр указывает на тесное взаимодействие между всеми сотрудниками.
- **Пример:** Если максимальное расстояние между двумя сотрудниками равно 5, диаметр графа равен 5.

---

#### **7. Кластеризация (Clustering Coefficient)**
- **Определение:** Мера того, насколько узлы в графе стремятся образовывать кластеры (группы).
- **Формула:**  
  \[
  \text{Clustering Coefficient}(v) = \frac{\text{Количество связей между соседями } v}{\text{Максимальное возможное количество связей между соседями } v}
  \]
- **Интерпретация:**
  - Высокий коэффициент кластеризации указывает на тесные группы сотрудников.
  - Низкий коэффициент кластеризации указывает на слабые связи внутри групп.
- **Пример:** Если у сотрудника Д 3 соседа, и они все связаны между собой, коэффициент кластеризации равен 1.

---

### Пример использования метрик в Python:
```python
import networkx as nx

# Создание графа
G = nx.DiGraph()
G.add_edges_from([(1, 2), (2, 3), (3, 1), (4, 5), (5, 4)])

# Расчет метрик
degree_centrality = nx.degree_centrality(G)
closeness_centrality = nx.closeness_centrality(G)
betweenness_centrality = nx.betweenness_centrality(G)
eigenvector_centrality = nx.eigenvector_centrality(G)
density = nx.density(G)
diameter = nx.diameter(G)
clustering = nx.clustering(G)

# Вывод результатов
print("Степень узла:", degree_centrality)
print("Центральность по близости:", closeness_centrality)
print("Центральность по посредничеству:", betweenness_centrality)
print("Центральность по собственному вектору:", eigenvector_centrality)
print("Плотность графа:", density)
print("Диаметр графа:", diameter)
print("Коэффициент кластеризации:", clustering)
```

---

### Заключение:
Эти метрики помогают:
1. Выявить ключевых сотрудников (экспертов и связных).
2. Определить "узкие места" в передаче знаний.
3. Оценить структуру сети и уровень взаимодействия между сотрудниками.

Использование этих метрик позволяет принимать обоснованные решения для улучшения процессов управления знаниями в организации.
